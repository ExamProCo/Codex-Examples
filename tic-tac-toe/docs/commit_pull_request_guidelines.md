# Commit & Pull Request Guidelines

Recent history shows short messages such as `update usage` and `reorganize`, but it is inconsistent. Use brief, imperative commit subjects that explain the change, for example `fix replay prompt handling`. Avoid vague messages like `rwarwraw`.

Pull requests should include:

- A short description of behavior changes.
- The commands used for validation.
- Terminal screenshots or pasted output when UI text or colors change.
